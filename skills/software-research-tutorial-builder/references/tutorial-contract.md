# Tutorial Contract

Use this file when building the final tutorial package.

## Required Sections

Every tutorial should contain these sections in this order:

1. What this software is for
2. When to use it and when not to
3. Environment and prerequisites
4. Installation or setup
5. First runnable example
6. Practical workflow example
7. Common mistakes and troubleshooting
8. Extension paths and further reading

## Required Companion Outputs

Add these supporting artifacts:
- a research brief,
- a support-material checklist,
- command snippets,
- config fragments when relevant,
- and a verification summary.

## Verification Summary

State separately:
- what was executed,
- what was only source-backed,
- and what remains unresolved.

## Writing Rules

- Use exact versions when available.
- Use explicit prerequisites.
- Prefer copy-paste-safe commands.
- Keep assumptions visible.
- Keep troubleshooting tied to concrete symptoms.

## Support-Material Checklist

At minimum, decide whether the package needs:
- environment variable examples,
- starter project structure,
- sample config files,
- sample input or output files,
- and cleanup steps.
