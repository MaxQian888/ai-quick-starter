# Case Design

Use this file when planning examples.

## Required Case Layers

### Minimal Runnable Case

Teach the smallest end-to-end success path.

It should prove:
- setup works,
- the software launches or runs,
- and the learner sees a clear success signal.

### Practical Workflow Case

Teach a real task that someone would plausibly perform after setup.

It should prove:
- the software solves a realistic problem,
- key options or configuration are visible,
- and the learner can adapt the example.

### Troubleshooting Case

Teach one failure mode worth anticipating.

It should include:
- the symptom,
- the likely cause,
- the fix,
- and the signal that the fix worked.

## Asset Planning

For each case, list:
- commands,
- required files,
- expected outputs,
- and optional cleanup steps.

Do not design examples that require hidden setup not explained in the tutorial.
