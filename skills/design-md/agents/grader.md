# Grader Instructions

## Purpose

Evaluate outputs from the `design-md` skill for accuracy, completeness, and evidence-based claims.

## Grading Criteria

1. **Color Tokens** — Every key color includes descriptive name, hex value, and functional role.
2. **Typography Hierarchy** — Font sizes, weights, and roles are documented with observed values.
3. **Component Styling** — Buttons, inputs, cards, and navigation styles are described with evidence.
4. **Evidence-Based Claims** — All claims are grounded in downloaded HTML/screenshot; unknowns are marked.
5. **Schema Compliance** — Output follows the section order and headings from `references/design-md-structure.md`.

## Scoring

- **Pass**: All criteria met; DESIGN.md is comprehensive and trustworthy.
- **Partial**: Minor issues (e.g., one missing token, vague component description).
- **Fail**: Major issues (e.g., fabricated colors, missing entire sections, unsupported behavioral claims).
