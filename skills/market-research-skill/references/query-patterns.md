# Query Patterns for Market Research

Use focused queries instead of broad terms. Combine an entity, a segment, and a business signal.

## Pattern Library

1. Demand signal
- `"<problem> software" + "adoption" + "<segment>"`
- `"<problem> + buy vs build + <segment>"`

2. Competitive movement
- `"<competitor>" + "pricing" + "2025 OR 2026"`
- `"<competitor>" + "launch" OR "new feature"`

3. Funding and expansion
- `"<industry> startup" + "funding" + "Series A OR Series B"`
- `"<company>" + "partnership" + "<region>"`

4. Compliance and policy
- `"<industry>" + "regulation" + "<region>"`
- `"<industry>" + "compliance requirement" + "<country>"`

5. Customer pain and alternatives
- `"<workflow>" + "manual process cost"`
- `"<problem>" + "alternatives" + "comparison"`

## Query Design Rules

1. Keep each query tied to one hypothesis.
2. Use 2-6 queries per scan to preserve signal quality.
3. Add timeframe terms (`2025`, `2026`, `when:30d`) for recency-sensitive research.
4. Add geography or language qualifiers when the target market is local.
5. Replace vague terms (`platform`, `solution`) with specific jobs-to-be-done.

## Example Query Sets

### AI Coding Tools (Enterprise)

1. `enterprise ai coding assistant pricing`
2. `secure on-prem code generation enterprise`
3. `developer productivity tool series b funding`
4. `ai code assistant compliance policy`

### Vertical SaaS (Logistics)

1. `warehouse orchestration software pricing`
2. `logistics automation startup funding 2026`
3. `warehouse labor shortage automation demand`
4. `warehouse saas implementation case study`

### SMB Fintech Operations

1. `smb expense management software alternatives`
2. `ap automation saas pricing small business`
3. `bookkeeping automation startup partnership`
4. `small business finance software churn reasons`
