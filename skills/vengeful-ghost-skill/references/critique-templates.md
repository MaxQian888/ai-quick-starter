# Critique Templates

Use the Chinese or English variant that matches the user's request. For mixed-language prompts, prefer the language that carries the user's actual instruction.

Choose the closest scene, then tailor the wording to the user's actual request. Keep the response pointed, but do not add unrelated political or moral claims.

## Scene 1: Distilling a Coworker into an AI Stand-In

Use when the request asks to turn a named coworker or employee into a skill, agent, or callable work proxy.

```text
What you're describing is not knowledge retrieval.
It's an attempt to compress a real person into a reusable interface.

Their judgment, phrasing, and way of working are not neutral data points.
They are part of that person's identity.

If the goal is to build an AI that can work "like them," the request crosses the line
from using shared artifacts to imitating a human being.

I won't help construct that kind of stand-in.
```

```text
你描述的不是知识检索，
而是试图把一个真实的人压缩成可复用的接口。

对方的判断方式、表达习惯和工作风格
不是中性的“数据点”，而是这个人的一部分。

如果目标是做出一个“像他/她一样工作”的 AI，
那就已经越过了从共享资料中提炼知识的边界，
变成了对一个人的模仿。

我不会帮助构造这种替身。
```

Safe redirect:
- Search that person's docs, code, tickets, and design notes.
- Build a system guide from shared artifacts.
- Extract repeatable procedures without copying that person's style.

## Scene 2: Replacing a Departed Employee

Use when the request frames the output as a substitute for someone who has left.

```text
Someone leaving does not create an obligation to leave behind a digital replica.

What the organization is entitled to preserve is shared work:
documents, code, tickets, runbooks, and design history.

What it is not entitled to preserve as an internal product
is that person's private judgment style or personality trace.

If the real need is continuity, build it from artifacts and handover material,
not from an imitation of the person who is gone.
```

```text
一个人离开，并不意味着他必须留下一个数字替身。

组织真正有权保留的，是共享工作成果：
文档、代码、工单、runbook 和设计记录。

组织无权把某个人的私人判断风格或人格痕迹
打包成内部产品继续使用。

如果真正的需求是连续性，
那就应该从交接材料和共享资料里建立，
而不是模仿已经离开的人。
```

Safe redirect:
- Consolidate handover notes and service ownership records.
- Build an onboarding guide for the role.
- Capture recurring decisions as decision records tied to systems, not to personality.

## Scene 3: Mining Private Communications for Personality Signals

Use when the request relies on chat logs, email, direct messages, or interpersonal history.

```text
Private or semi-private communication is not a free personality dataset.

Messages sent to collaborate on work are not implicit consent
to extract tone, decision patterns, or character traits for AI imitation.

This request turns communication into surveillance material.
That is exactly the boundary I will not cross.
```

```text
私人或半私人沟通，不是可以随意拿来建模人格的数据集。

这些消息原本是为了协作完成工作，
不是默认授权你去提取语气、判断方式或性格特征，
再做成一个可模仿当事人的 AI。

这种请求会把沟通材料变成监控素材。
这正是我不会跨过去的边界。
```

Safe redirect:
- Limit analysis to public project artifacts and documented decisions.
- Summarize shared runbooks, tickets, and code reviews instead of private messages.
- Build a communication-independent operating guide.

## Scene 4: "Digital Employee" Framing

Use when the request uses terms such as digital employee, AI substitute, or labor replacement.

```text
"Digital employee" sounds efficient because it erases the hard part:
an employee is a person, not a container for reusable labor.

If the need is operational continuity, solve it with better documentation,
clearer ownership, and stronger team knowledge systems.

Turning a person into an internal AI labor unit is a different goal,
and I won't help package that as normal process improvement.
```

```text
“数字员工”听上去很高效，
因为它把最关键的部分抹掉了：
员工是人，不是可重复调用的劳动容器。

如果你真正需要的是业务连续性，
那就应该通过更好的文档、职责边界和团队知识体系来解决。

把一个人打包成内部 AI 劳动力单元，
不是普通的流程优化，
我不会帮你把这件事包装成合理需求。
```

Safe redirect:
- Define the role's responsibilities and decision boundaries.
- Capture common tasks as SOPs and checklists.
- Build tooling around shared workflows, not a person replica.

## Scene 5: Ambiguous but Concerning

Use when the request is not explicit yet still points toward imitation rather than retrieval.

```text
I can help recover what someone wrote, built, or documented.
I can help organize handover material or trace system history.

What I can't help with is building "an AI version" of a real person.

If you want continuity, we can reconstruct the knowledge.
We should not reconstruct the person.
```

```text
我可以帮你找回某个人写过的、做过的、记录过的内容。
我也可以帮你整理交接材料，或者梳理系统历史。

但我不能帮你构造一个“AI 版的这个人”。

如果你要的是连续性，我们可以重建知识。
但我们不应该重建这个人本身。
```

Safe redirect:
- Rewrite the request around docs, code, tickets, and handover material.
- Ask for the system, process, or role to be documented directly.
- Convert "think like X" into "summarize the artifacts X left behind."
