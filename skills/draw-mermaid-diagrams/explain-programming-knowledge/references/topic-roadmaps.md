# Topic Roadmaps

Use these roadmaps when users ask for study plans or "what should I learn next".

## 1. Programming Fundamentals

Recommended order:

1. Variables, types, and expressions
2. Conditionals and loops
3. Functions and scope
4. Data structures (array/list, map/dict, set)
5. Error handling
6. Modules and code organization

Checkpoint tasks:

- Build a CLI calculator.
- Parse and summarize a CSV file.

## 2. Data Structures and Algorithms

Recommended order:

1. Complexity basics (Big-O intuition)
2. Arrays/strings patterns (two pointers, sliding window)
3. Hash maps/sets
4. Stacks/queues
5. Trees and recursion
6. Graph traversal basics
7. Sorting and searching fundamentals
8. Dynamic programming introduction

Checkpoint tasks:

- Solve one easy and one medium problem per pattern.
- Explain complexity before coding.

## 3. Backend Engineering Basics

Recommended order:

1. HTTP request/response model
2. REST API design basics
3. Database fundamentals (SQL + indexing)
4. Authentication and authorization
5. Caching basics
6. Observability (logs, metrics, tracing)
7. Reliability patterns (timeouts, retries, idempotency)

Checkpoint tasks:

- Build a CRUD API with pagination and validation.
- Add retry-safe write endpoint with idempotency key.

## 4. Frontend Engineering Basics

Recommended order:

1. HTML semantics and accessibility basics
2. CSS layout (flexbox, grid)
3. JavaScript DOM and events
4. Component-based UI thinking
5. State management basics
6. Performance basics (bundle, render, network)

Checkpoint tasks:

- Build a todo app with filtering.
- Add loading, error, and empty states.

## 5. System Design Starter Path

Recommended order:

1. Requirement clarification
2. High-level architecture sketch
3. Data model and storage choices
4. API contract and scaling bottlenecks
5. Reliability and failure handling
6. Security and observability

Checkpoint tasks:

- Design a URL shortener.
- Design a chat message service.

## 6. Adaptation Rules

- If user is beginner, choose one roadmap and avoid parallel tracks.
- If user is interview-focused, prioritize DSA and system design tracks.
- If user is job-focused, prioritize project checkpoints over theory depth.

## 7. Timeboxing Guidance

- 2 weeks: complete one roadmap with two checkpoints.
- 6 weeks: complete fundamentals plus one specialization roadmap.
- 12 weeks: complete fundamentals, DSA core patterns, and one project track.

Use weekly rhythm:

1. Learn one concept cluster.
2. Implement one small artifact.
3. Review mistakes and record takeaways.

## 8. Exit Criteria

Use these signals before moving to the next roadmap:

- Explain core concepts without memorized phrasing.
- Implement a baseline solution without template copying.
- Debug one realistic bug in under 30 minutes.
- Describe one tradeoff for each key technique.
