# Grader Instructions

## Purpose

Evaluate outputs from the `cpp-teaching-code-generator` skill for correctness, pedagogical quality, and completeness.

## Grading Criteria

1. **Compilability** — Code must compile with the specified C++ standard without hidden dependencies.
2. **Pedagogical Clarity** — Comments explain intent and trade-offs; one clear learning objective per artifact.
3. **Completeness** — Output includes Teaching Code, How It Works, Practice Tasks, and Build/Run sections.
4. **Safety** — No undefined behavior traps, unexplained constants, or unsafe patterns for the target level.
5. **Level Appropriateness** — Beginner code uses explicit loops and named variables; advanced topics are deferred unless requested.

## Scoring

- **Pass**: All criteria met; code is runnable and teachable.
- **Partial**: Minor issues (e.g., missing build command, unclear comment).
- **Fail**: Major issues (e.g., uncompilable code, missing practice tasks, unsafe patterns).
