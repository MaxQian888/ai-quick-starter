# 配图打法

知乎好回答几乎都配图。这里讲**怎么决定每张图、用哪条路子做出来**。配图的作用和节奏见 [zhihu-high-score-patterns.md 的"配图节奏"](zhihu-high-score-patterns.md#配图节奏)——这里只管落地。

## 先定方案，再做图

在【确认④ 配图方案】这一步，针对正文逐段过一遍，为每张图定四件事，做成一张表给用户确认：

| # | 放在哪（段落/小标题） | 画什么（内容） | 类型 | 出图方式 |
|---|---|---|---|---|
| 图1 | 开头，作封面 | 一句话标题 + 主视觉 | 封面图 | gpt-image 生成 |
| 图2 | "怎么选"这节后 | 3 款产品 5 维度对比 | 对比表/信息图 | gpt-image 生成 |
| 图3 | "数据"这节 | 近 5 年 XX 增长曲线 | 数据图表 | 推荐网图（带来源） |

不要先闷头生成一堆图，**先把这张表给用户拍板**——图比文字贵，返工成本高。

## 两条出图路子，怎么选

### 路子 A：gpt-image 真生成（首选用于示意/信息/封面/概念图）

适合：封面图、信息图、流程图、概念示意、对比示意、梗图风格图——**不需要真实精确数据、靠设计取胜**的图。

调用方式：**直接调用 `gpt-image` skill**（这台机器装了）。把"画什么"翻译成一段清晰的中文/英文 prompt 交给它。要点：
- 描述要具体：主体、风格、版式、文字内容、配色、画幅。知乎封面常用竖图或 16:9 横图。
- 图上若要中文字，明确写出要出现的文字，并说明"中文、排版清晰"。gpt-image 对中文排版支持较好，但仍要把字数控制少、给准。
- 信息图/对比图：把要呈现的条目、维度、数值都写进 prompt，让它按"干净的信息图版式"排。
- 生成后看一眼是否达意，不对就改 prompt 重生成，别将就。

> gpt-image skill 会自查它自带的 Reference Gallery 找匹配的设计范式，再调它打包的 CLI/脚本出图。把需求讲清楚交给它即可，不要自己另写生图代码。

### 路子 B：推荐网上现成图（用于需要真实精确数据/实物/截图的图）

适合：真实统计图表、真实产品实拍、新闻截图、需要权威出处的数据图——**这类图自己生成会失真或造假，必须用真图**。

做法：
- 用 WebSearch / WebFetch 找现成图，给出**图片链接 + 来源页面链接**，让用户自行判断版权后使用。
- 明确标注"建议自行核实版权/时效后使用"，不要把网图当作已授权直接嵌入成品。
- 找不到合适的真图时，退而求其次：用 gpt-image 生成"示意图"，并在图注里注明"示意，非真实数据"。

## 在正文里怎么标注

成品 markdown 里，每张图用图片占位 + 图注的形式落位，方便用户替换/上传到知乎：

```markdown
![封面：3 步搞定 XX（gpt-image 已生成，文件：cover.png）](cover.png)

> 图注：XX 的三个关键步骤一图看懂

![近 5 年 XX 市场规模（建议用真实数据图，来源见下）](图片待插入)

> 图注：数据来源 https://example.com/report ，请自行核实时效与版权
```

gpt-image 真生成出来的，给出文件路径让用户直接用；推荐网图的，给链接和图注，标清来源与版权提示。

## 别犯的错
- 图海凑数、贴无关美图——伤完读率，也显得不用心。
- 把网图当真实数据图直接嵌入却不标来源——版权 + 真实性双重风险。
- gpt-image 生成"假装真实"的统计图表配上编造数字——这是造假，不要做；真实数据走路子 B。
