# custom：用户提供样本，反推风格

参考：Nina Panickssery 的 "few-shot fake conversation" 方法、ACL 2022 "Recipe for Arbitrary Text Style Transfer"。

## 何时用 custom

- 用户的写作风格鲜明，3 个预设都不贴合
- 用户想模仿某个作者（自己、朋友、某位作家）
- 用户给出"我以前写的稿子"作为锚

## 三步走

### Step 1：让用户贴 1-2 段样本

最好 100-500 字。问用户：

> 请贴 1-2 段你喜欢的中文写作样本（你自己的稿子或你想模仿的作者）。要 100 字以上，最好是完整段落。

### Step 2：反推风格 + 反推 prompt

读样本后，你要做**两件事**：

**A. 描述这段文字的风格（5-8 句话）**

- 句长有多大波动？
- 段长偏好？
- 人称偏好（第一 / 第二 / 第三）？
- 有没有口语词 / 方言 / 网络词？
- 修辞偏好（直陈 / 比喻 / 反问 / 对比 / 自嘲）？
- 节奏偏好（密集 / 松散 / 突变）？
- 情感温度（冷静 / 热烈 / 自嘲）？
- 标志性的句式或词汇？

**B. 反推"如果让一个 LLM 写出这段，prompt 会怎么写"**

这一步是 Panickssery 方法的核心：把样本变成"假对话历史"。

例子：

用户样本：
> "我那天加班到凌晨。Slack 没人。咖啡冷了。Code review 还有 8 个。我盯着屏幕看了 5 分钟，关了电脑。回家路上想，这是不是就是大家说的 burnout。"

你反推：

```
风格描述：
- 第一人称、私人视角
- 短句多（"Slack 没人。咖啡冷了。"）
- 用具体细节代替情绪词（不说"我很累"，说"咖啡冷了"）
- 反高潮收尾（不抒情、不下结论，留疑问）
- 节奏：每句一个动作 / 一个观察

反推 prompt：
"用第一人称，写你某次加班到凌晨的具体场景。
用短句。每句一个具体动作或观察（不要直接说情绪）。
结尾要克制——留一个没回答的小问题，不要升华，不要总结。"
```

### Step 3：基于反推的 prompt + 风格描述做重写

把"风格描述 + 反推 prompt + 用户原文"组合成系统提示给自己，按这个风格改写。

注意：

- **不要照抄样本的具体内容**——抄结构和节奏
- **不要复制样本的具体词汇**——除非是用户的特征词
- **抓住"重复出现的特征"**——一个人的风格在于他的重复，不在于他的偶然

## 反 anti-pattern

不要做：

- 把样本的修辞硬塞到原文（"加班"样本是凌晨场景，原文是产品介绍——不能强行也变成凌晨）
- 用样本的具体名词（样本里有"Slack"，原文里没有就别加）
- 模仿到 1:1 像（这是抄袭不是 style transfer，目标是"读起来像同一作者"，不是"读起来像这一段"）

## 实操检查

重写后问自己：

1. 如果把样本和重写后的文字混在一起，能不能看出是同一作者？
2. 重写有没有保留原文的**信息**？（不能为了风格丢内容）
3. 有没有把样本的具体细节（地点、时间、名字）误抄进重写？

3 项都通过才算合格。
