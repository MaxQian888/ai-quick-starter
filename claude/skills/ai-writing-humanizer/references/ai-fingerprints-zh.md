# 中文 AI 写作指纹词表

按命中扣分严重程度分层：**硬指纹**（命中即重写）→ **软警告**（视语境保留）。
来源：Humanizer-zh、知乎降 AI 率实测、Anthropic / 知乎实证。

## 一、过渡 / 连接词（硬指纹）

最强 AI 信号，命中必删或改写。

| AI 套话 | 替代 / 改法 |
|--------|------------|
| 此外 | 删除，下一句直接讲；或换成"另外"+ 具体内容 |
| 综上所述 / 总而言之 / 综上 | 删除；如果真要总结，用一个具体观察代替 |
| 值得注意的是 / 值得一提的是 | 删除引导词，直接讲内容 |
| 不可否认 | 删除；改成"确实" 或直接陈述 |
| 众所周知 | 删除；如果真是常识没必要说，反之要给依据 |
| 与此同时 | 改成"同时"（一字差感觉差很多） |
| 首先 / 其次 / 再次 / 最后 | 用句子节奏取代编号；只在确实需要时保留 1-2 个 |
| 在这个 X 的时代 / 在...的背景下 | 删除开场套话，直接讲 X |
| 综合来看 / 综合分析 | 删除；具体说哪个角度 |

## 二、关联结构（硬指纹）

完美对仗结构是 AI 标志，要打破。

| 套话结构 | 改法 |
|---------|------|
| 不仅 X 而且 Y | 拆成两句："X。Y 也一样。" |
| 既 X 又 Y | 选一个最重要的；或换成"X，同时 Y" |
| 一方面 X，另一方面 Y | 删掉前缀，直接陈述两面 |
| 无论 X 还是 Y，都 Z | 拆成两句陈述 |
| 在 X 的同时（也）Y | "X，也 Y"——把"同时"删了 |
| X 与 Y 相辅相成 | 替换成具体关系："X 让 Y 更准 / X 是 Y 的前提" |

## 三、夸大 / 模糊归因（硬指纹）

| 套话 | 问题 | 改法 |
|------|------|------|
| 广受好评 | 谁评的？多广？ | 给具体来源："App Store 4.8 / 知乎 1.2k 赞" |
| 深受 X 喜爱 | 同上 | 给具体场景或数据 |
| 引发广泛关注 | 同上 | "微博 #X# 当晚 500 万阅读" |
| 具有重要意义 | 空话 | 删除；如果真重要，说具体后果 |
| 产生深远影响 | 空话 | 同上 |
| 备受瞩目 | 同上 | 同上 |
| 业内人士表示 / 专家指出 | 假权威 | 删除；或给真名字和来源 |

## 四、空话收尾（硬指纹）

公众号 AI 文的标志结尾，必删。

- 挑战与机遇并存
- 未来可期
- 让我们拭目以待
- 相信在不久的将来
- 让我们一起 X
- 希望以上内容对您有所帮助
- 如有任何问题，欢迎随时告诉我
- 截至我所了解的信息 / 根据我现有的知识

**替代收尾**：用具体观察、未解决的问题、或一个反高潮的小细节。

## 五、强行名词化 / 动词化（软警告）

中文 AI 喜欢把动词变名词词组（"做出 X" / "进行 X"），稀释信息。

| 软词组 | 改 |
|--------|---|
| 做出了一个决定 | 决定了 |
| 进行了讨论 | 讨论了 |
| 给出了解释 | 解释了 |
| 加以解决 | 解决 |
| 进行优化 | 优化 |
| 做出贡献 | 贡献 |
| 实现了 X 的目标 | 达到了 X |
| 起到 X 作用 | 是 X / X-ed |

强制规则：**`进行` / `做出` / `加以` / `实现 ... 的目标` 这种结构能删就删**。

## 六、限定词膨胀（软警告）

中文里"水蛭"：删了不影响意思。

```
非常 其实 基本上 实际上 大概 相对而言 总体而言
应该说 可以说 在某种程度上 一定意义上 在某种意义上
确实 的确 当然 显然 必然 必须 务必
```

**规则**：每段最多保留 1 个，连续 3 个删 2 个。

## 七、句法 / 结构指纹（硬指纹）

不是词，是结构。

1. **三段式硬套**：开头"X 的背景与意义" + 中间"分析与举例" + 结尾"挑战与展望"——拆掉这种骨架
2. **三联词强迫症**：每段都"高效、稳定、可靠"或"快速、安全、便捷"——三选一
3. **加粗滥用**：把非关键词加粗（"价值"、"增长"这种）——只加粗关键名词
4. **行内标题强迫症**：每个二级思路都"**标题**：内容"——只在真有 2+ 个并列子标题时用
5. **句长方差低**：连续 3 句字数差 < 5——故意混合长短句
6. **段首单调**：80% 段落以主语开头——变换段首（用问句、单字句、时间状语、地点状语开头）

## 八、标点指纹（硬指纹）

| 信号 | 阈值 |
|------|------|
| em-dash `—` 单段超过 1 次 | 立刻命中 |
| em-dash 全文超过 1 / 200 字 | 命中 |
| 弯引号 `""` 混在普通中文里 | 命中（粘贴自富文本痕迹） |
| 中文混用全角半角逗号 | 警告 |
| 标题用 Title Case（如"Build A Better App"） | 不该出现在中文文章 |

## 九、语义层指纹

不是词、不是句法，是"读完整段没获得任何具体信息"。

- 全文 0 个数字
- 全文 0 个专有名词（人名 / 公司名 / 产品名 / 工具名 / 地点）
- 全文 0 个第一人称（"我 / 我们" 或类似）
- 通篇中立、毫无作者立场
- 段段都"先肯定再否定"完美平衡

**如果以上 ≥ 3 个命中，记为"严重缺人味"。**

## 十、奉承尾巴（硬指纹）

ChatGPT 出厂味最浓的部分：

- "希望以上内容对您有所帮助 / 希望对您有所启发"
- "如有任何问题，欢迎随时告诉我"
- "祝您工作顺利 / 学习愉快"
- "感谢您的阅读"

**规则**：文章结尾任何讨好性句子都删。
