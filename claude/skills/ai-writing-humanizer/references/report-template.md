# 诊断报告 Markdown 模板

每次诊断都按这个结构出报告。空缺项写"未命中"或省略该小节，不要假装有问题。

## 完整模板

```markdown
# 诊断报告：<文章标题或前 20 字>

**语种**：zh / en / mixed
**字数**：N
**总评**：[高 / 中 / 低] AI 浓度（基于硬指纹命中数）

---

## 一、硬指纹（高置信度 AI 痕迹）

| 类别 | 命中内容 | 位置 |
|------|---------|------|
| 中文套话 | "综上所述" | §2 末段 |
| 关联结构 | "不仅 X 而且 Y" | §1 第 3 句 |
| em-dash 滥用 | — 共 4 次（阈值 ≤ 1 / 200 字） | §1, §3 |
| 三联词 | "高效、稳定、可靠" | §3 |
| 奉承尾巴 | "希望对您有所启发" | 末段 |

## 二、信息密度问题

按"修改前 → 修改后"列出，每条带行号。

- **§1 第 2 句**："实际上这其实是个非常重要的问题" → "这是个关键问题"（删 6 字）
- **§3 第 1 句**："做出了一个决定" → "决定了"
- **§4 全段**：被动语态，可改主动
- **§5 第 3 句**：副词堆叠"非常迅速地完成"→"迅速完成"或"几分钟完成"

## 三、人味缺失

- 全文 0 个具体名词（缺工具名 / 品牌名 / 数字 / 人名）
- 句长标准差 2.3（建议 ≥ 4，需要混合长短句）
- 段首 87% 以主语开头，多样性不足
- 零作者立场（通篇中立陈述）
- 缺微观察或具体故事

## 四、风格初判

基于词汇、结构、语气，倾向 **wechat-zh** 模式（节奏画面型）。

**理由**：
- 文体偏散文 / 叙事
- 词汇偏口语
- 当前结构问题主要是太工整，需要打破对仗

## 五、建议下一步

- [ ] 用户选择改写模式：technical-zh / wechat-zh / marketing-zh / custom
- [ ] 是否需要重写？如果只要诊断不重写，告诉我

---
```

## 字段说明

### 总评的判定阈值

| 总评 | 硬指纹命中数 | 信息密度问题数 | 人味缺失项 |
|------|------------|---------------|----------|
| 高 AI 浓度 | ≥ 5 | ≥ 8 | ≥ 4 项中 ≥ 3 |
| 中 AI 浓度 | 2-4 | 3-7 | 4 项中 ≥ 2 |
| 低 AI 浓度 | ≤ 1 | ≤ 2 | 4 项中 ≤ 1 |

### "位置"如何标

文本短（< 500 字）：直接用句子序号或段落序号
文本中（500-2000 字）：用 "§N 第 M 句"
文本长（> 2000 字）：用脚本的行号

### 简化版（用户只要快速判定时）

如果用户只想知道"AI 味重不重"，给精简版：

```markdown
# 快速诊断

**AI 浓度**：高 / 中 / 低
**核心问题**（最多 3 条）：
1. 中文套话 5 处（综上所述、不仅...而且、值得注意的是）
2. 句长太均匀（σ=2.3，建议 ≥ 4）
3. 零具体名词

**建议**：完整改写 / 局部修补 / 不需要改
```

## 报告写作规则

1. **不要堆砌**：命中 1 个就列 1 个，不要为了报告"完整"假装有问题
2. **行号定位准**：让用户能直接定位到原文
3. **建议要具体**：不要"建议优化语言"——说"建议把 §2 的'综上所述'删了"
4. **不要重复**：同一个 AI 词出现 5 次只列一次（标"共 5 次"）
5. **总评要克制**：原文真的不像 AI 就直说，不要为了显示 skill 在干活硬挑刺
