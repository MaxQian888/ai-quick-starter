# AI Writing Humanizer

中文为主、兼顾英文的写作 skill，专门解决三大问题：

1. **信息密度低**——废话多、限定词堆叠、抽象动词代替具体动词
2. **表达奇怪**——AI 套话指纹（"综上所述"、"不仅...而且"、"delve into"、em-dash 滥用）
3. **缺人味**——零具体名词、零立场、句长方差低、完美对称结构

## 何时使用本 skill

强触发词（一旦出现就用本 skill，不要犹豫）：

- "改一下这段"、"重写"、"精简"、"降 AI 率"、"让它有人味"、"太空了"、"读起来像 AI"、"像机器写的"、"不像人话"
- "写公众号"、"写技术博客"、"写营销文案"、"写推文"、"写软文"
- "诊断这段"、"分析下这篇"、"看看哪里 AI 味重"
- 命令式：`/humanize`、`/de-ai`、`/降AI`

弱信号（也应该启动本 skill）：

- 用户抱怨自己写的稿子"工整但没意思"、"逻辑对但没记忆点"
- 用户让你写任何**面向真人读者**的中长篇内容（不是写代码注释、API 文档）

## 工作流：4 步精修

```
[Step 1] 诊断 (scripts + references)
    ↓
[Step 2] 询问用户：选模式 + 是否需要重写
    ↓
[Step 3] 针对性重写（仅修诊断出的问题，不机械全删）
    ↓
[Step 4] 输出主要改动 3-5 条 + 为什么改
```

下面是每一步的具体做法。

### Step 1：诊断扫描

**先跑硬指纹扫描脚本（便宜准确）：**

```bash
python scripts/scan_fingerprints.py --text "<用户输入文本>" --lang zh
# 或 --lang en，或 --lang auto 自动判断
```

脚本会输出 JSON：

```json
{
  "lang": "zh",
  "stats": {"chars": 850, "sentences": 18, "sentence_length_stddev": 2.3},
  "hits": [
    {"category": "中文套话", "term": "综上所述", "line": 7, "context": "..."},
    {"category": "em-dash 滥用", "count": 4, "threshold": "≤ 1 / 200 字"},
    {"category": "三联词", "term": "高效、稳定、可靠", "line": 3}
  ],
  "structural_warnings": [
    "句长标准差 2.3，低于 4，缺乏 burstiness",
    "段首 80% 以主语开头，多样性不足"
  ]
}
```

**再做语义层诊断（脚本管不了的）：**

读以下 reference 文件，按里面的规则人工判断：

- `references/ai-fingerprints-zh.md` 或 `references/ai-fingerprints-en.md`——按语言选
- `references/density-rules.md`——信息密度 12 条
- `references/voice-injection.md`——人味 8 条

诊断重点：

- 是否有具体名词、数字、专有名词？（如果通篇抽象——记为缺人味）
- 是否有作者立场？（缺 "我认为/我觉得/但我看来"——记为零立场）
- 是否有微观察或具体细节？（如果只有通论——记为缺人味）
- 句子是否过度对称？（开头 80% 都"主语 + 动词 + 宾语"——结构单调）

**生成报告**（参考 `references/report-template.md`）：

```markdown
# 诊断报告

## 一、硬指纹（高置信度 AI 痕迹）
| 类别 | 命中 | 位置 |
|------|------|------|
| 中文套话 | "综上所述" | §2 末段 |
| em-dash 滥用 | — 共 4 次 | §1 §3 |
| 三联词 | "高效、稳定、可靠" | §3 |

## 二、信息密度问题
- §1 第 2 句"实际上这其实是个非常重要的问题"→ "这是个关键问题"（删 6 字）
- §3 第 1 句"做出了一个决定"→ "决定了"
- §4 全段被动语态可改主动

## 三、人味缺失
- 全文 0 个具体名词（缺工具名/品牌名/数字）
- 句长标准差 2.3（建议 ≥ 4）
- 零作者立场

## 四、风格初判
基于词汇和结构，倾向 **wechat-zh** 模式（节奏画面型）
```

### Step 2：询问用户选模式

**关键：必须问用户，不要擅自决定。**

问法（中文）：

> 我已经扫出 N 处问题。请选一个改写模式：
>
> 1. **technical-zh**——简洁思辨型（技术博客、科普、深度文章）。保留专业准确，去学术腔，敢用具体例子和立场。
> 2. **wechat-zh**——节奏画面型（公众号、自媒体）。短句多、第一/第二人称、加微观察、敢用句子片段。
> 3. **marketing-zh**——高能带感型（营销、文案、推广）。金句 + 画面 + 行动号召，反高潮。
> 4. **custom**——你贴 1-2 段你喜欢的样本，我反推出风格再改。
>
> 选哪个？或者只要诊断不要改？

英文文本对应 `technical-en` / `casual-en` / `marketing-en` / `custom`。

**如果用户选 custom**：让用户贴样本 → 读 `references/voice-profiles/custom-sample.md` 按 Panickssery 法处理。

### Step 3：针对性重写

读对应的 voice profile（如 `references/voice-profiles/wechat-zh.md`），然后**只针对 Step 1 诊断出的问题改写**，不要机械应用所有规则。

重写时必须遵守的硬约束：

- **不为了改而改**：原文如果某句已经很好（具体、有立场、节奏好），原样保留
- **不替换同义词糊弄**：把 "delve into" 换成 "look into" 不算改，要改句法和结构
- **不全禁 em-dash**：控频率即可（中文 ≤ 1/200 字，英文 ≤ 1/200 词）
- **不强加 typo 或感叹号**：业界确认这两招让文本更假
- **不强行全部缩写**："it's" 全替 "it is" 会矫枉过正
- **强制注入**：至少 1 个具体名词/数字/工具名/品牌名/时间——这是"有人味"的最强信号

如果原文确实抽象到没有任何可注入的具体细节（如纯哲学讨论），明确告诉用户："原文缺少可锚定的具体事实，建议补充一个例子，我可以根据你提供的例子改"。

### Step 4：解释主要改动

输出 3-5 条核心改动，每条 1 句话说"改了什么 + 为什么"：

```markdown
## 主要改动

1. 删了 §1 三处"实际上其实"——空话不传递信息（删字 = 提密度）
2. §2 末段"综上所述"开头 → 直接用观察句——AI 总结词指纹强
3. §3 "高效稳定可靠"三联词 → "稳定"——三选一最准
4. §4 加了"凌晨 3 点的 stack trace 第 47 行"——具体细节补人味
5. §5 第一人称改写——给立场比中立陈述更有可信度
```

## 触发模式总览

| 用户意图 | Skill 行为 |
|---------|-----------|
| "诊断这段" | 只做 Step 1，给报告 |
| "改一下这段"（无指定模式） | 完整 4 步 |
| "/humanize <text>" | 完整 4 步 |
| "写一篇关于 X 的公众号" | **写之前**读 wechat-zh 注入约束，写完做 Step 1 自检 |
| "写一篇关于 X 的技术博客" | 同上，用 technical-zh |
| "写一段营销文案" | 同上，用 marketing-zh |

## Anti-patterns（业界已证伪，不要做）

1. **同义词替换表**（"furthermore → also"）——检测器看节奏不看词
2. **全局加 typo**——真人 typo 有特定模式，假 typo 反而更可疑
3. **全局缩写**（"it is"→"it's"）——真人缩写不均匀
4. **加感叹号 / 全大写**——读起来像营销小广告
5. **全禁 em-dash**——真人也用，控频率即可
6. **追求"反检测器"**——目标是"像深思熟虑的真人"，不是"骗过 GPTZero"
