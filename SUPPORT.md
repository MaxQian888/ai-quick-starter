# Support / 支持

## Where To Ask For Help / 在哪里寻求帮助

Use the following routes so requests land in the right place:

- Bug report / 缺陷反馈:
  Open a GitHub issue with the bug report template.
- Feature request / 功能建议:
  Open a GitHub issue with the feature request template.
- Security issue / 安全问题:
  Follow [SECURITY.md](SECURITY.md) and report privately.
- Usage question / 使用问题:
  Open an issue and include the skill path, expected behavior, and what you already tried.

请按下列方式提交问题，方便更快分流：

- 缺陷反馈：
  使用 bug report 模板提交 issue。
- 功能建议：
  使用 feature request 模板提交 issue。
- 安全问题：
  按 [SECURITY.md](SECURITY.md) 私下报告。
- 使用问题：
  提交 issue 时请附上技能路径、预期行为以及你已经尝试过的步骤。

## What Makes Support Faster / 怎样能更快得到帮助

- Link the exact file or skill path
- Include relevant logs or screenshots
- Mention whether the issue is local-only or reproducible by others
- Share tool versions when scripts are involved

- 给出准确的文件或技能路径
- 附上相关日志或截图
- 说明问题是仅本地出现还是他人也可复现
- 如果涉及脚本，请提供工具版本信息

## Scope / 支持边界

This repository shares reusable skills and supporting assets. Maintainers may not be able to provide custom consulting, private debugging, or service-specific support for every third-party integration.

本仓库提供的是可复用技能与配套资产。维护者未必能为每个第三方集成提供定制咨询、私有排障或长期支持。
