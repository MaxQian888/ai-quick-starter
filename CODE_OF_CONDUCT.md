# Code Of Conduct / 行为准则

`ai-quick-starter` aims to be a practical, respectful, and low-drama open repository for reusable AI skills.

`ai-quick-starter` 希望成为一个务实、尊重他人、低摩擦的公开 AI 技能仓库。

## Our Standards / 我们的标准

Examples of behavior that help build a positive environment:

- Being respectful in technical disagreement
- Giving actionable feedback instead of personal attacks
- Sharing context, evidence, and reproducible details
- Welcoming contributors with different backgrounds and skill levels
- Respecting maintainers' time, scope decisions, and repository boundaries

有助于建立良好社区氛围的行为包括：

- 在技术分歧中依然保持尊重
- 提供可执行的反馈，而不是人身攻击
- 分享上下文、证据和可复现细节
- 欢迎不同背景和不同经验水平的贡献者
- 尊重维护者的时间、范围判断和仓库边界

Examples of unacceptable behavior:

- Harassment, insults, or intimidation
- Hate speech or discriminatory language
- Repeated bad-faith arguing, trolling, or spamming
- Publishing private information without permission
- Knowingly submitting malware, secrets, or destructive content

不可接受的行为包括：

- 骚扰、侮辱、威胁或恐吓
- 仇恨言论或歧视性表达
- 持续恶意争论、钓鱼、刷屏
- 未经允许公开他人隐私信息
- 明知故犯地提交恶意代码、密钥或破坏性内容

## Scope / 适用范围

This code of conduct applies to:

- GitHub issues, pull requests, and review comments
- Repository discussions around skills, scripts, and documentation
- Public interactions that are directly connected to this project

本准则适用于：

- GitHub issues、pull requests 和评审评论
- 围绕技能、脚本和文档的仓库协作
- 与本项目直接相关的公开互动场景

## Enforcement / 执行方式

The maintainer may remove comments, reject contributions, lock threads, or ban participants whose behavior violates this policy.

维护者有权删除评论、拒绝贡献、锁定讨论，或限制违反本准则的参与者。

Enforcement decisions will try to balance fairness, repository safety, and contributor trust.

执行时会尽量兼顾公平性、仓库安全和贡献者信任。
